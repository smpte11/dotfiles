# Pure-inspired yazi flavor — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a `pure.yazi` flavor for yazi whose UI inherits the terminal's ANSI palette (Pure-style) and whose preview syntax theme is templated from the chezmoi `active_theme`.

**Architecture:** A chezmoi-managed yazi config under `dot_config/yazi/`. `theme.toml` selects the flavor. `flavor.toml` is static and uses only ANSI color names + `reset`, so the whole UI follows the live ghostty palette with no templating. `tmtheme.xml.tmpl` is the single templated file; it renders hex from `active_theme` in `.chezmoidata.toml`.

**Tech Stack:** yazi 26.5.6, chezmoi (Go templates), TOML, tmTheme (Apple plist XML), python3 `tomllib`, `xmllint`, `plutil`.

## Global Constraints

- `flavor.toml` MUST contain **no hex colors** — only `reset` and the 17 yazi-allowed ANSI names: `reset, black, white, red, lightred, green, lightgreen, yellow, lightyellow, blue, lightblue, magenta, lightmagenta, cyan, lightcyan, gray, darkgray`.
- Only `tmtheme.xml.tmpl` is a chezmoi template; every other file is static (no `.tmpl` suffix).
- yazi has **no headless mode** — it grabs the tty and blocks. Never run bare `yazi` in an automated step; the only yazi-launch check is a manual one performed by the user.
- **Do not run any `git` or `jj` command.** Each task ends at a "ready to commit" checkpoint; the user commits.
- Pure → ANSI mapping (authoritative for all UI styling):
  - `magenta` = primary accent (normal mode, selection markers, active tab, find position, dialog active/hovered)
  - `blue` = path / `cwd` / directories
  - `darkgray` = muted chrome (borders, inactive tabs, secondary status, symlinks)
  - `yellow` = progress / executables-adjacent / select mode
  - `red` = errors / orphan symlinks / cut markers
  - `lightmagenta` = secondary accent (unset mode, copy/find tints)
  - `reset` = file listing body / dialog body text

A minimal flavor inspired by the [Pure](https://github.com/sindresorhus/pure)

`flavor.toml` uses **only ANSI color names** (`magenta`, `blue`, `darkgray`, …)

- [ ] **Step 1: Write the failing validator**

Create `scratchpad/check_flavor.py`:

```python
import tomllib, pathlib, sys

PATH = pathlib.Path("dot_config/yazi/flavors/pure.yazi/flavor.toml")
ALLOWED = {
    "reset","black","white","red","lightred","green","lightgreen",
    "yellow","lightyellow","blue","lightblue","magenta","lightmagenta",
    "cyan","lightcyan","gray","darkgray",
}

raw = PATH.read_text()
assert "#" not in raw, "flavor.toml must contain no hex colors (found '#')"

data = tomllib.loads(raw)

bad = []
def walk(node):
    if isinstance(node, dict):
        for k, v in node.items():
            if k in ("fg", "bg") and isinstance(v, str):
                if v not in ALLOWED:
                    bad.append((k, v))
            else:
                walk(v)
    elif isinstance(node, list):
        for x in node:
            walk(x)
walk(data)
assert not bad, f"non-ANSI fg/bg values: {bad}"

for section in ("mgr","tabs","mode","status","which","filetype"):
    assert section in data, f"missing [{section}] section"
assert data["mgr"]["cwd"]["fg"] == "blue"
assert data["mode"]["normal_main"]["bg"] == "magenta"
print("OK")
```
